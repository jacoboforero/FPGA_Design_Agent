# Glossary

Last verified against runtime behavior: March 8, 2026.

This glossary defines key terms used across engineer and researcher documentation.

## User Roles
- **Hardware Engineer**: primary user focused on design intent to usable RTL artifacts via the interactive/full CLI flow.
- **Researcher**: primary user focused on systematic benchmarking, reproducibility, and model/system comparison.

## Run Concepts
- **Preset**: named runtime behavior profile in `config/runtime.yaml` (for example `engineer_fast`, `benchmark`).
- **Run Name**: logical run label used for observability tracking.
- **Run ID**: unique ID associated with a specific execution instance.
- **Campaign**: grouping label for related benchmark runs.
- **Run Root (Run Directory)**: benchmark run directory, typically `<output_root>/<campaign>/<run_id>/`.

## Benchmark Concepts
- **Canonical**: low-variance benchmark profile (default `n=1`).
- **Sampled**: higher-variance multi-sample benchmark profile (config-driven `n`, temperature, top_p).
- **Mode Directory**: benchmark output directory for one profile (`canonical/` or `sampled/`).
- **Official Artifacts**: `summary.txt` and `summary.csv` generated by official analyzer flow.
- **Aggregate**: local structured convenience summary (`aggregate.json`) parsed from official artifacts.
- **Run Manifest**: run-level metadata file (`run_manifest.json`) containing config/model/flags/filter context.

## Pipeline Concepts
- **Planner**: stage that converts frozen specs into `design_context.json` and `dag.json`.
- **Orchestrator**: runtime service that drives node execution across stage state transitions.
- **Repair Loop**: bounded retry path through distill/reflect/debug on failures.
- **Task Memory**: staged artifact workspace under `artifacts/task_memory/`.

## Artifact Concepts
- **Generated Artifacts**: outputs under `artifacts/generated/`, including planned and emitted design files.
- **Observability Artifacts**: telemetry and metrics under `artifacts/observability/runs/<run_name>/<run_id>/...`.
- **Index Artifacts**: searchable CSV/JSON outputs under `artifacts/index/` from `scripts/index_runs.py`.
- **Organized View**: symlinked campaign-oriented browsing tree under `artifacts/organized/`.

## Safety and Control Flags (Benchmark)
- `--resume`: continue run by skipping already-produced sample artifacts.
- `--overwrite`: replace existing run directory contents.
- `--dry-run`: show run plan only, do not execute generation.
- `--purge-queues`: optional queue purge before benchmark execution (unsafe in shared environments).
