Module: event_logger_top
Title: Dual-channel timestamped event logger with AXI-Lite control and FIFO

L1
Role summary: Dual-channel event logger that timestamps incoming events and stores them in a FIFO exposed via AXI-Lite; raises an interrupt on overflow or threshold.
Key rules:
- Capture an event when cfg_enable=1, eventX_valid=1, and (eventX_code & cfg_maskX) == cfg_maskX.
- If both channels are valid in the same cycle and FIFO has space, select round-robin using last_grant; after reset, channel0 has priority.
- Event word format (64b): [63:32]=timestamp, [31]=src_id (0=event0, 1=event1), [30:23]=code, [22:7]=data, [6:0]=0.
- If FIFO is full, drop the selected event and set overflow_sticky=1.
- overflow_sticky clears only on cfg_clear_overflow pulse.
- irq asserted when cfg_irq_enable=1 and (overflow_sticky=1 or fifo_level >= cfg_threshold). If cfg_threshold=0, irq depends only on overflow_sticky.
Performance intent: Accept at most one event per cycle; FIFO supports simultaneous read/write.
Reset semantics: Active-low async reset clears FIFO, timestamp, irq, overflow_sticky, and registers; cfg_enable=0, cfg_mask0/1=0, cfg_threshold=0.
Corner cases:
- Both event_valids high in same cycle.
- FIFO full during event capture.
- cfg_threshold=0 and cfg_maskX=0.
- cfg_enable toggles mid-stream.
- irq_enable toggles while overflow_sticky is set.
- event_valid pulses during reset assertion/deassertion.
Open questions:
- none

L3
Test goals:
- Correct event word formatting with timestamp and source ID.
- Round-robin arbitration when both channels assert valid.
- Overflow behavior and sticky flag handling.
- IRQ assertion and deassertion based on threshold and overflow.
- Correct AXI-Lite register access and FIFO read behavior.
Oracle strategy: Cycle-accurate reference model for timestamp, FIFO, overflow_sticky, and irq.
Stimulus strategy: Randomized event_valid pulses on both channels with random codes/data, mixed with directed AXI-Lite register writes/reads and mid-stream resets.
Pass/fail criteria:
- Event words match reference format and ordering.
- FIFO level matches reference; no underflow.
- overflow_sticky and irq match reference each cycle.
- AXI-Lite reads/writes obey protocol and return expected data.
